Instructions de Déploiement GitHub
Formulaire LAB TGS France

1. Introduction

Ce document détaille toutes les étapes nécessaires pour déployer le formulaire LAB (Lutte Anti-Blanchiment) de TGS France sur GitHub Pages. Le formulaire sera accessible publiquement via une URL permanente et fonctionnera de manière autonome avec envoi automatique des PDF par email.

Important : Ce déploiement permettra à vos clients d'accéder au formulaire 24h/24 et 7j/7, avec envoi automatique des formulaires complétés à votre adresse email.

2. Prérequis

Avant de commencer, assurez-vous d'avoir :

  • Un compte GitHub actif (gratuit)
  • Accès à vos fichiers dans l'espace de stockage AI Drive
  • Une connexion internet stable
  • Un navigateur web moderne

3. Structure des Fichiers à Télécharger

Voici les fichiers que vous devez récupérer depuis votre espace de stockage :

Fichier Description Obligatoire
questionnaire_lab_logo_tgs_france.html Formulaire principal avec logo TGS France Oui

3.1. Téléchargement des Fichiers

  1. Accédez à votre espace de stockage : https://www.genspark.ai/aidrive/files/formulaires_clients
  2. Téléchargez le fichier questionnaire_lab_logo_tgs_france.html
  3. Sauvegardez-le sur votre ordinateur

4. Création du Repository GitHub

4.1. Création du Repository

  1. Connectez-vous à votre compte GitHub
  2. Cliquez sur le bouton vert "New" ou l'icône "+" en haut à droite
  3. Sélectionnez "New repository"
  4. Configurez le repository :
    • Repository name : questionnaire-lab
    • Description : "Formulaire LAB TGS France"
    • ✅ Cochez "Public"
    • ✅ Cochez "Add a README file"
  5. Cliquez sur "Create repository"

4.2. Upload du Fichier HTML

  1. Dans votre nouveau repository, cliquez sur "Add file"
  2. Sélectionnez "Upload files"
  3. Glissez-déposez ou sélectionnez le fichier questionnaire_lab_logo_tgs_france.html
  4. IMPORTANT : Renommez le fichier en index.html avant de le commiter
  5. Ajoutez un message de commit : "Ajout du formulaire LAB TGS France"
  6. Cliquez sur "Commit changes"
Attention : Le fichier DOIT s'appeler index.html pour que GitHub Pages l'affiche automatiquement à la racine de votre site.

5. Configuration de GitHub Pages

5.1. Activation de GitHub Pages

  1. Dans votre repository, cliquez sur l'onglet "Settings"
  2. Faites défiler vers le bas jusqu'à la section "Pages"
  3. Dans "Source", sélectionnez "Deploy from a branch"
  4. Dans "Branch", sélectionnez "main"
  5. Laissez "/ (root)" sélectionné
  6. Cliquez sur "Save"

5.2. Obtention de l'URL Publique

Après avoir sauvegardé, GitHub vous indiquera l'URL de votre site :

https://[votre-nom-utilisateur].github.io/questionnaire-lab/

Note : Il peut y avoir un délai de 5 à 10 minutes avant que le site soit accessible.

6. Vérifications à Effectuer

6.1. Test du Formulaire

  1. Accédez à votre URL GitHub Pages
  2. Vérifiez que le logo TGS France s'affiche correctement
  3. Testez le remplissage de quelques champs
  4. Vérifiez la signature électronique
  5. Testez l'export PDF
  6. Vérifiez l'envoi email (avec un formulaire test)

6.2. Liste de Vérification

7. Partage du Formulaire

7.1. URL à Communiquer

Une fois le déploiement réussi, communiquez cette URL à vos clients :

https://[votre-nom-utilisateur].github.io/questionnaire-lab/

7.2. Communication Clients

Exemple de message pour vos clients :

"Dans le cadre de nos obligations légales de lutte anti-blanchiment, nous vous invitons à compléter notre questionnaire en ligne à l'adresse suivante : [votre-url]. Ce formulaire sécurisé vous permettra de saisir vos informations et de signer électroniquement le document."

8. Maintenance et Mises à Jour

8.1. Modification du Formulaire

Pour modifier le formulaire :

  1. Téléchargez la nouvelle version depuis AI Drive
  2. Renommez-la en index.html
  3. Uploadez-la dans votre repository GitHub (remplace l'ancienne)
  4. Commitez les changes

8.2. Surveillance des Emails

Vérifiez régulièrement votre boîte email benjamin.guillet@tgs-france.fr pour les formulaires reçus.

9. Dépannage

9.1. Problèmes Courants

Problème Solution
Site non accessible Vérifier que GitHub Pages est activé et attendre 10 minutes
Logo ne s'affiche pas Vérifier la connexion internet du client
Email non reçu Vérifier les spams et tester avec un autre email
Formulaire ne se valide pas Vérifier que tous les champs obligatoires sont remplis
